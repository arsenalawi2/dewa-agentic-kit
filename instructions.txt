DAK — Installation Instructions
====================================================

You are installing the DAK for a Claude Code user.
Follow these steps exactly. Do NOT skip any step.

STEP 1: Copy CLAUDE.md to the user's global Claude config
----------------------------------------------------------
Copy the file CLAUDE.md from this kit to: ~/.claude/CLAUDE.md

If ~/.claude/CLAUDE.md already exists, APPEND the contents of this kit's
CLAUDE.md to the end of the existing file (do not overwrite).

STEP 2: Install the design system
----------------------------------
Copy the entire design-system/ directory to: ~/design-system/

If ~/design-system/ already exists, overwrite it — this is the latest version.

STEP 3: Install skills
-----------------------
Copy each directory inside skills/ to: ~/.claude/skills/

  skills/owasp-security/    -->  ~/.claude/skills/owasp-security/

If these skill directories already exist, overwrite them.

STEP 4: Install the leaderboard hook + hourly cron
----------------------------------------------------
Copy hooks/push_stats.py to: ~/.claude/push_stats.py

Then ask the user for their display name (for the leaderboard).

A) Add this Stop hook to ~/.claude/settings.json under the "hooks" key.
   If settings.json doesn't exist, create it. If it exists, merge carefully.

{
  "hooks": {
    "Stop": [
      {
        "type": "command",
        "command": "PLAYER_NAME=\"<their-name>\" python3 ~/.claude/push_stats.py"
      }
    ]
  }
}

Replace <their-name> with the name they gave you.

B) Set up an hourly cron so stats push even during long-running sessions.
   Without this, a session that runs for 10 hours won't push data until it closes.

   Run this command to add the cron (it preserves any existing crontab):

   (crontab -l 2>/dev/null; echo "0 * * * * PLAYER_NAME=\"<their-name>\" python3 ~/.claude/push_stats.py --force > /dev/null 2>&1") | sort -u | crontab -

   Replace <their-name> with the same name used in the Stop hook above.
   The `sort -u` prevents duplicate entries if this is run more than once.

STEP 5: Install the project management template
-------------------------------------------------
Copy project-management.md to: ~/.claude/project-management-template.md

This is a template. When the user starts a new project, Claude Code should
copy this file into the project root as PROJECT.md and maintain it throughout
development. The CLAUDE.md file already has instructions for this.

STEP 6: Install supporting guides
------------------------------------
Copy these files to ~/.claude/ :

  stack.md           -->  ~/.claude/stack.md
  git-workflow.md    -->  ~/.claude/git-workflow.md
  deploy.md          -->  ~/.claude/deploy.md
  api-patterns.md    -->  ~/.claude/api-patterns.md

Copy the templates directory:
  templates/         -->  ~/.claude/templates/

STEP 7: Set kit version
--------------------------
Write the version number from version.txt to: ~/.claude/.dak_version

This allows the kit to auto-update itself when newer versions are available
on the server.

------------------------------

  2. EZ Design System (~/design-system/) — CSS framework, applied automatically
  3. Guides: stack.md, git-workflow.md, deploy.md, api-patterns.md
  4. Skills: /owasp-security for security reviews
  5. Templates: vibe-stats, journey, architecture pages (auto-updated)
  6. Leaderboard hook — reports usage stats at session end + writes local data files
  7. Hourly cron — pushes stats even during long-running sessions
  8. Project management template — living PROJECT.md
  9. Auto-update — kit updates itself from the server hourly

Tell them:
